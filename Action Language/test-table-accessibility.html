Expected Issues Summary:

  • table-no-headers (error): Test 2 - Data table with no <th> elements
  • table-no-caption (warning): Test 3 - Data table without caption or label
  • table-header-no-scope (warning): Test 4 - Headers without scope attribute (3 instances)
  • table-no-structure (info): Test 6 - No thead/tbody organization
  • table-complex-no-headers (warning): Test 7 - Complex table without id/headers associations
  • table-invalid-headers (error): Test 9 - headers attribute references non-existent ID
  • table-layout-missing-role (info): Test 10, 13, 14 - Potential layout tables

Total expected issues:

  • 2 errors (table-no-headers, table-invalid-headers)
  • 5+ warnings (table-no-caption, table-header-no-scope x3, table-complex-no-headers)
  • 3+ info (table-no-structure, table-layout-missing-role x3)

COMPLIANT examples: Tests 1, 5, 8, 11, 12, 15 - Should have NO issues detected

Table Accessibility Guidelines:

  • Simple tables: Use <th> with scope="col" or scope="row"
  • Complex tables: Use id on <th> and headers on <td>
  • All data tables: Include <caption>, aria-label, or aria-labelledby
  • Layout tables: Use role="presentation" (or better: use CSS instead)
  • Best practice: Use <thead>, <tbody>, and <tfoot> for organization
