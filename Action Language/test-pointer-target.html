WCAG Target Size Requirements:

  • WCAG 2.5.5 (Level AAA): 44√ó44 CSS pixels (recommended best practice)
  • WCAG 2.5.8 (Level AA, WCAG 2.2): 24√ó24 CSS pixels (minimum requirement)
  • Spacing: At least 8px between adjacent interactive elements (or targets exceed size requirements)

Exceptions:

  • Inline text links: Links within sentences are exempt
  • Essential: Target size is essential to the information being conveyed
  • User control: User can change the target size (e.g., browser zoom)
  • Equivalent: Another control on the page provides equivalent functionality with adequate size

Why Target Size Matters:

  • Motor disabilities: Users with tremors or limited fine motor control struggle with small targets
  • Age-related factors: Older users often have reduced dexterity
  • Situational limitations: Using device while moving, one-handed operation
  • Touch screens: Finger size makes small targets difficult to tap accurately
  • Assistive technology: Users with stylus, mouth stick, or head pointer benefit from larger targets

Design Tips:

  • Start with 44√ó44px minimum for all interactive elements
  • Add padding to increase tappable area without changing visual size
  • Use flexbox/grid gap for spacing between adjacent targets
  • Enlarge checkboxes and radio buttons (default size is often too small)
  • Expand hit area using pseudo-elements when visual size must stay small
  • Ensure adequate line-height for inline links (1.5-1.6 minimum)
  • Test on actual touch devices with different finger sizes
  • Consider touch-action: manipulation to remove 300ms tap delay

Testing:

  • Measure interactive elements in browser DevTools
  • Test with actual touch devices (phones, tablets)
  • Try tapping targets with different finger sizes
  • Check spacing between adjacent buttons and links
  • Verify checkboxes and radio buttons are large enough
  • Test with motor disability simulators
