ARIA Switch Requirements

Essential Requirements for Accessible Switches

  • role="switch" on the control element (or use <input type="checkbox" role="switch">)
  • aria-checked="true" or aria-checked="false" to indicate state (NOT the checked attribute)
  • aria-label or aria-labelledby to provide an accessible name
  • Space and Enter keys must toggle the switch state
  • Must be keyboard focusable (tabindex="0" on custom elements or use native button/input)
  • Visual focus indicator must be clearly visible
  • State changes must be announced to screen readers

⌨️ Keyboard Interactions

Space Toggle the switch between on and off Enter Toggle the switch between on and off (for button-based switches) Tab Move focus to the next focusable element Shift + Tab Move focus to the previous focusable element

Accessibility Comparison

✓ Accessible Switch

  • ✓ Has role="switch"
  • ✓ Has aria-checked state
  • ✓ Keyboard accessible (Space/Enter)
  • ✓ Has accessible label
  • ✓ Screen readers announce state
  • ✓ Visible focus indicator
  • ✓ Works with assistive tech

✗ Inaccessible Switch

  • ✗ Missing role="switch"
  • ✗ No aria-checked state
  • ✗ Not keyboard accessible
  • ✗ No accessible label
  • ✗ Silent to screen readers
  • ✗ No focus indicator
  • ✗ Mouse-only operation
