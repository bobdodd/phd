Slider Pattern

Interactive demonstration of accessible vs inaccessible slider implementations

About the Slider Pattern

A slider is an input widget that allows users to select a value from a range by moving a handle along a track. Sliders can be used for various purposes like volume control, brightness adjustment, selecting ranges, and more.

ARIA Requirements:

  • role="slider" on the interactive element
  • aria-label or aria-labelledby for accessible name
  • aria-valuemin for minimum value
  • aria-valuemax for maximum value
  • aria-valuenow for current value
  • aria-valuetext (optional) for formatted value display
  • aria-orientation for vertical sliders (default is horizontal)
  • tabindex="0" for keyboard accessibility

Keyboard Support:

  • Arrow Right/Up: Increase value by one step
  • Arrow Left/Down: Decrease value by one step
  • Home: Set to minimum value
  • End: Set to maximum value
  • Page Up: Increase by larger increment (optional)
  • Page Down: Decrease by larger increment (optional)

5 Accessibility Issues Detected

  • Issue 1: Missing role="slider" The inaccessible sliders lack role="slider", so screen readers cannot identify them as interactive slider controls.
  • Issue 2: No accessible name Missing aria-label or aria-labelledby means screen reader users don't know what the slider controls.
  • Issue 3: Missing ARIA value attributes Without aria-valuemin, aria-valuemax, and aria-valuenow, assistive technologies cannot announce the current value or range.
  • Issue 4: No keyboard support Missing tabindex="0" and keyboard event handlers prevent keyboard-only users from interacting with the sliders.
  • Issue 5: Missing aria-orientation for vertical slider The vertical slider lacks aria-orientation="vertical", causing confusion about which arrow keys to use.

Testing Instructions

Keyboard Testing

  1. Press Tab to navigate to the accessible sliders - they should receive focus with a visible indicator
  2. Use Arrow Right/Up to increase the value and Arrow Left/Down to decrease
  3. Press Home to jump to minimum value and End to jump to maximum
  4. Try Page Up and Page Down for larger increments on the brightness slider
  5. Try tabbing to the inaccessible sliders - they won't receive focus

Screen Reader Testing

  1. Enable your screen reader (NVDA, JAWS, VoiceOver, etc.)
  2. Navigate to the accessible sliders - they should announce as "slider" with the label and current value
  3. Adjust the value - the screen reader should announce the new value
  4. Navigate to the inaccessible sliders - they won't be announced as interactive controls
  5. The accessible temperature slider should read the valuetext (e.g., "68 degrees Fahrenheit")

Mouse Testing

  1. Click and drag slider thumbs - both accessible and inaccessible versions should work
  2. Click on the track to jump to a position
  3. Verify that focus indicators appear when clicking on accessible sliders
