Detecting Problematic Tabindex Patterns

The Problem: Chaotic Tab Order

The tabindex attribute controls keyboard navigation order. When used incorrectly, it creates confusing, illogical focus flows that disorient keyboard users. Paradise's FocusOrderConflictAnalyzer detects common problematic patterns.

WCAG requires focus order to be meaningful and predictable. Arbitrary positive tabindex values, duplicate values, and extreme jumps all violate this requirement.

Common Focus Order Problems

1. Duplicate Tabindex Values

<input tabindex="1">
<input tabindex="1"> <!-- Same! -->
<input tabindex="2">

Issue: Unpredictable order when multiple elements share same tabindex

2. Large Gaps in Sequence

<input tabindex="1">
<input tabindex="100"> <!-- Jump! -->
<input tabindex="101">

Issue: Large gaps suggest maintenance problem, potential missing elements

3. Non-Sequential Values

<input tabindex="1">
<input tabindex="3"> <!-- Skips 2 -->
<input tabindex="5"> <!-- Skips 4 -->

Issue: Gaps in sequence create confusion, may indicate deleted elements

4. Reverse Order

<input tabindex="3"> <!-- Backward! -->
<input tabindex="2">
<input tabindex="1">

Issue: Focus flows opposite to visual/logical order

Best Practices

‚úÖ Recommended: Don't Use Positive Tabindex

The best solution is to avoid positive tabindex values entirely. Use DOM order for focus flow, which is predictable and maintainable.

<!-- Good: Natural DOM order -->
<input placeholder="Name">
<input placeholder="Email">
<input placeholder="Phone">
<button>Submit</button>

When you need custom order: Restructure your HTML instead of using tabindex. Focus order should match visual order, which should match DOM order.

Valid Uses of Tabindex

  • tabindex="0": Add element to natural tab order (for custom widgets)
  • tabindex="-1": Remove from tab order but allow programmatic focus (for skip links, modals)
  • Positive values: Only if absolutely necessary, use sequential values (1, 2, 3...)

WCAG 2.1 Compliance

Focus order problems violate:

  • 2.4.3 Focus Order (Level A): Navigation sequence must preserve meaning and operability. Chaotic tabindex breaks this.
  • 2.4.7 Focus Visible (Level AA): Focus must be visible and predictable. Unexpected jumps violate predictability.
  • 3.2.1 On Focus (Level A): Focusing an element shouldn't cause unexpected changes. Backward focus feels unexpected.
