Focus Management Demo

Proper handling of focus when removing or hiding elements

About This Pattern

When removing or hiding focused elements, you must check if they have focus and move focus elsewhere. Otherwise, keyboard users lose their place in the page.

  • WCAG Criteria: 2.4.3 (Focus Order), 2.4.7 (Focus Visible)
  • Key Principle: Always check document.activeElement before removing/hiding
  • Common Mistakes: element.remove(), style.display='none', .blur() without moving focus

// Scenario 1: Removing focused element
function removeElement(element) {
  // Check if element (or any child) has focus
  if (element.contains(document.activeElement)) {
    // Move focus to a safe location first
    const nextFocusable = findNextFocusable(element);
    if (nextFocusable) {
      nextFocusable.focus();
    } else {
      document.body.focus();
    }
  }

  // Now safe to remove
  element.remove();
}

// Scenario 2: Hiding focused element
function hideElement(element) {
  // Check if element (or any child) has focus
  if (element.contains(document.activeElement)) {
    // Move focus before hiding
    const toggleButton = document.getElementById('toggle');
    toggleButton.focus();
  }

  // Now safe to hide
  element.setAttribute('hidden', '');
}

// Helper function
function findNextFocusable(currentElement) {
  const focusable = document.querySelectorAll(
    'button, [href], input, select, textarea, [tabindex]:not([tabindex="-1"])'
  );

  for (let i = 0; i < focusable.length; i++) {
    if (!currentElement.contains(focusable[i])) {
      return focusable[i];
    }
  }
  return null;
}

  // Missing: Check if element.contains(document.activeElement)

// Other problematic patterns:
function blurWithoutReplacement() {
  // ISSUE: Calling blur() without moving focus elsewhere
  element.blur(); // ← Standalone blur() detected!
}

function removeParent() {
  // ISSUE: Removing parent of focused element
  parent.remove(); // ← Still detected if child has focus
}

Issues the analyzer will detect:

  • removal-without-focus-management - element.remove() without checking document.activeElement (WCAG 2.4.3, 2.4.7)
  • hiding-focused-element - style.display='none' or setAttribute('hidden') on focused element
  • standalone-blur - Calling .blur() without moving focus to another element
  • focus-on-non-focusable-element - .focus() called on element without tabindex
  • positive-tabindex - tabindex > 0 disrupts natural tab order
