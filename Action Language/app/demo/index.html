About This Demo

This is a comprehensive educational tool showcasing the capabilities of the ActionLanguage accessibility analyzer. Each demo page presents side-by-side comparisons of accessible (good) and inaccessible (bad) implementations of common web patterns.

Use this demo to learn about web accessibility, understand WCAG requirements, and see how the analyzer detects issues in real code.

How to Use This Demo

  1. Browse: Click on any demo page to see examples
  2. Compare: Try both accessible and inaccessible versions with keyboard only (Tab, Enter, arrows)
  3. Analyze: Run the tool: npm run analyze demo/pages/buttons.html
  4. Learn: Compare detected issues with expected issues listed on each page
  5. Test: Open pages in VS Code to see diagnostics and use the "Apply Fix" feature

WCAG 2.1 Criteria Covered

  • 1.3.1 Info and Relationships - Semantic structure and ARIA
  • 2.1.1 Keyboard - All functionality available via keyboard
  • 2.1.2 No Keyboard Trap - Users can escape all components
  • 2.1.4 Character Key Shortcuts - Remap/disable single-key shortcuts
  • 2.4.3 Focus Order - Logical focus navigation
  • 2.4.7 Focus Visible - Visible focus indicator
  • 2.5.3 Label in Name - Visible labels match accessible names
  • 3.2.1 On Focus - No unexpected context changes on focus
  • 3.2.2 On Input - No unexpected context changes on input
  • 4.1.2 Name, Role, Value - Proper ARIA roles and attributes
  • 4.1.3 Status Messages - Announcements for dynamic updates
