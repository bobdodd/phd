About the Carousel Pattern

A carousel presents a rotating set of content items (usually images) one at a time. Carousels are commonly used for image galleries, product showcases, featured content, and promotional banners.

Accessibility Requirements

  • role="region" on the carousel container with aria-roledescription="carousel"
  • aria-label on the carousel region to identify it
  • role="group" with aria-roledescription="slide" on each slide
  • aria-label on each slide indicating position (e.g., "1 of 5")
  • Previous and Next buttons with proper aria-label attributes
  • Pause/Play button for controlling auto-rotation
  • Tab panel pattern for slide indicators with proper ARIA roles
  • Keyboard navigation with Arrow keys or Tab key
  • Auto-rotation stops on mouse hover or keyboard focus
  • Live region announcements for slide changes

Keyboard Interaction

  • Tab: Move focus through carousel controls (Previous, Next, Pause/Play, indicators)
  • Enter/Space: Activate focused button or indicator
  • Arrow Left/Right: Navigate between slide indicators (when focused)
  • Home/End: Move to first/last slide indicator (when focused)

Common Use Cases

  • Image Galleries: Showcasing photos in a slideshow format
  • Product Showcases: Highlighting featured products with images and details
  • Hero Banners: Rotating promotional content on landing pages
  • Testimonials: Displaying customer reviews one at a time
  • Featured Content: Rotating news articles or blog posts

Implementation Guide

HTML Structure

<div role="region" aria-roledescription="carousel" aria-label="Featured products">
  <div class="carousel-header">
    <h3>Featured Products</h3>
    <button class="play-pause" aria-label="Pause automatic slide rotation">
      Pause
    </button>
  </div>

  <div class="carousel-viewport">
    <div class="carousel-slides">
      <div role="group" aria-roledescription="slide" aria-label="1 of 4">
        <!-- Slide content -->
      </div>
    </div>

    <button class="prev" aria-label="View previous slide">‹</button>
    <button class="next" aria-label="View next slide">›</button>
  </div>

  <div role="tablist" aria-label="Slide navigation">
    <button role="tab" aria-label="Slide 1" aria-selected="true"></button>
    <button role="tab" aria-label="Slide 2" aria-selected="false"></button>
  </div>
</div>

<div aria-live="polite" aria-atomic="true" class="sr-only"></div>

Key Accessibility Features

  • Semantic Roles: Use role="region" with aria-roledescription="carousel" to identify the carousel
  • Slide Information: Each slide has role="group" with aria-roledescription="slide" and position information
  • Control Labels: All interactive elements have descriptive aria-label attributes
  • Auto-rotation Control: Provide Pause/Play button to stop automatic slide transitions
  • Pause on Interaction: Stop auto-rotation when user hovers or focuses within carousel
  • Tab Panel Pattern: Use role="tablist" and role="tab" for slide indicators
  • Keyboard Navigation: Support arrow keys for navigating between indicators
  • Live Announcements: Use aria-live="polite" region to announce slide changes

Testing Checklist

  • Screen reader announces carousel region and current slide
  • All controls are keyboard accessible with Tab key
  • Previous/Next buttons navigate slides correctly
  • Pause/Play button toggles auto-rotation
  • Auto-rotation pauses on hover and focus
  • Slide indicators work with Enter/Space and arrow keys
  • Focus indicators are clearly visible
  • Slide changes are announced to screen readers
