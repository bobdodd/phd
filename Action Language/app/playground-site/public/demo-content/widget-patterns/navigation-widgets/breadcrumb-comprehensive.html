Code Walkthrough

HTML Structure (Accessible)

<nav aria-label="Breadcrumb">
  <ol>
    <li>
      <a href="/home">Home</a>
      <span aria-hidden="true">&gt;</span>
    </li>
    <li>
      <a href="/products">Products</a>
      <span aria-hidden="true">&gt;</span>
    </li>
    <li aria-current="page">
      MacBook Pro 16"
    </li>
  </ol>
</nav>

Why This Structure Works

1. <nav> Landmark

<nav aria-label="Breadcrumb">
  <!-- Content -->
</nav>

<nav> creates a navigation landmark that screen readers recognize. aria-label="Breadcrumb" distinguishes it from other navigation (main menu, footer links, etc.).

2. Ordered List for Hierarchy

<ol>
  <li>Home</li>
  <li>Products</li>
  <li>MacBook Pro</li>
</ol>

<ol> (ordered list) communicates hierarchy to screen readers. They announce: "List, 3 items... 1 of 3... 2 of 3... 3 of 3". This helps users understand their depth in the navigation.

3. Links for Navigation

<li>
  <a href="/products">Products</a>
</li>

<a> elements are naturally keyboard accessible and announced as links. Current page is plain text (not a link to itself).

4. aria-current="page"

<li aria-current="page">
  MacBook Pro 16"
</li>

aria-current="page" explicitly marks the current page. Screen readers announce: "MacBook Pro 16 inch, current page".

5. Hidden Separators

<span aria-hidden="true">&gt;</span>

aria-hidden="true" hides visual separators from screen readers. They're purely decorative and would clutter announcements. Screen readers already announce the list structure ("1 of 5, 2 of 5...").
