Feed Pattern

Social media feed with infinite scroll, keyboard navigation, and proper ARIA

High Complexity WCAG 2.1.1, 2.4.3, 4.1.2, 4.1.3

What is a Feed?

A feed is a section containing a scrollable list of articles where scrolling may cause articles to be added or removed from either end. Feeds are commonly used in social media, news sites, and activity streams where content is loaded dynamically.

Complete Feed Pattern Requires:

  • role="feed" on container
  • role="article" on each feed item
  • aria-label or aria-labelledby on feed
  • aria-setsize on articles (total count)
  • aria-posinset on articles (position)
  • aria-busy="true" during loading
  • tabindex="0" on articles for keyboard focus
  • Page Up/Page Down: Navigate between articles
  • Ctrl+Home: Jump to first article
  • Ctrl+End: Jump to last loaded article
  • Focus management: New items announce position
  • Load more: Infinite scroll or explicit button

// Feed navigation implementation
const feed = document.getElementById('feed');
const articles = feed.querySelectorAll('[role="article"]');

// Page Down: Move to next article
feed.addEventListener('keydown', (event) => {
  const current = document.activeElement;
  const currentIndex = Array.from(articles)
    .indexOf(current);

  if (event.key === 'PageDown') {
    event.preventDefault();
    const nextIndex = Math.min(
      currentIndex + 1,
      articles.length - 1
    );
    articles[nextIndex].focus();

  } else if (event.key === 'PageUp') {
    event.preventDefault();
    const prevIndex = Math.max(currentIndex - 1, 0);
    articles[prevIndex].focus();

  } else if (event.ctrlKey && event.key === 'Home') {
    event.preventDefault();
    articles[0].focus();

  } else if (event.ctrlKey && event.key === 'End') {
    event.preventDefault();
    articles[articles.length - 1].focus();
  }
});

// Load more with aria-busy
async function loadMore() {
  feed.setAttribute('aria-busy', 'true');

  const newPosts = await fetchPosts();
  const currentSize = articles.length;

  newPosts.forEach((post, index) => {
    const article = createArticle(post);
    const position = currentSize + index + 1;

    article.setAttribute('aria-posinset', position);
    article.setAttribute('aria-setsize',
      currentSize + newPosts.length);

    feed.appendChild(article);
  });

  feed.setAttribute('aria-busy', 'false');

  // Update setsize on all articles
  const allArticles = feed.querySelectorAll('[role="article"]');
  const totalSize = allArticles.length;
  allArticles.forEach(article => {
    article.setAttribute('aria-setsize', totalSize);
  });
}

Why This Matters

Feed patterns are critical for accessibility in social media and news sites. Without proper ARIA attributes and keyboard navigation, screen reader users cannot understand the structure of the feed, don't know which article they're on, and cannot efficiently navigate content. The combination of role="feed", aria-setsize/posinset, and Page Up/Down navigation provides the equivalent experience for keyboard users as scrolling does for mouse users.
