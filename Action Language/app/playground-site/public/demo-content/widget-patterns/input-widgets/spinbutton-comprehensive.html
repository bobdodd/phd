Spinbutton Pattern

Interactive number input controls with keyboard navigation

ARIA Spinbutton Pattern Requirements

  • role="spinbutton" on the input element
  • aria-valuemin, aria-valuemax, aria-valuenow attributes
  • aria-label or aria-labelledby for accessible naming
  • aria-valuetext (optional) for custom value descriptions
  • Arrow Up/Down to increment/decrement value
  • Home/End keys to jump to min/max values
  • Page Up/Down for larger increments (optional)

Paradise Detected Issues (5)

  • Issue 1: Missing role="spinbutton" Screen readers cannot identify these as spinbutton controls. Users don't know they can use arrow keys or what type of input is expected.
  • Issue 2: Missing aria-valuemin, aria-valuemax, aria-valuenow No information about current value or valid range. Screen readers cannot announce the current state or bounds to users.
  • Issue 3: Missing accessible labels Inputs lack proper aria-label or aria-labelledby. Screen reader users don't know what each spinbutton controls.
  • Issue 4: No keyboard navigation Arrow keys, Home, End, and Page Up/Down don't work. Keyboard users cannot operate the controls efficiently.
  • Issue 5: Missing aria-valuetext For non-numeric displays (like "16px" or "50%"), screen readers read the formatted text incorrectly without proper value context.
