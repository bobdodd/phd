Autocomplete Syntax:


autocomplete=""


autocomplete="shipping street-address"
autocomplete="billing cc-number"


autocomplete="section-primary name"
autocomplete="section-primary email"


autocomplete="[section-*] [billing|shipping] "

Benefits of Autocomplete:

  • Faster form filling: Users can complete forms in seconds instead of minutes
  • Fewer errors: Reduces typos in addresses, emails, phone numbers
  • Better accessibility: Critical for users with cognitive disabilities, motor impairments
  • Mobile friendly: Much easier than typing on mobile keyboards
  • Password managers: Enables auto-login and secure password generation
  • Compliance: Required by WCAG 2.1 Level AA (1.3.5)

When autocomplete="off" is OK:

  • Search boxes (though consider keeping search history)
  • One-time codes (use "one-time-code" instead)
  • Truly sensitive data where autofill creates security risk
  • Fields that change meaning based on context

Note: Modern browsers often ignore autocomplete="off" on password fields because password managers are critical for security. Use autocomplete="new-password" or "current-password" instead.
